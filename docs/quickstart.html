Getting Started

Quick Start

TransNovel을 설치하고, LLM 제공자를 연결한 뒤, 첫 일본 웹소설 챕터를 한국어로 번역하는 최소 경로입니다.

시작하기 최신 릴리즈
준비물 설치 LLM 설정 첫 번역 관심작품 문제 해결

Step 0

준비물

TransNovel 자체는 무료 오픈소스 앱입니다. 실제 번역에는 Gemini API 키가 필요합니다.

처음 테스트할 때는 짧은 챕터 하나로 모델, 비용, 번역 톤을 확인한 뒤 긴 챕터를 번역하세요.
  • macOS Apple Silicon 또는 Windows x64/ARM64 환경
  • Google 계정
  • Google AI Studio에서 발급한 Gemini API 키
  • 번역할 지원 사이트의 챕터 URL

Step 1

앱 설치

  1. 최신 릴리즈에서 운영체제에 맞는 설치 파일을 받습니다.
  2. macOS는 .dmg, Windows는 setup.exe 파일을 실행합니다.
  3. 앱을 실행하고 상단 탭에서 설정으로 이동합니다.

macOS

처음 실행 시 확인되지 않은 개발자 경고가 뜰 수 있습니다. 시스템 설정의 개인 정보 보호 및 보안에서 실행을 허용하세요.

Windows

SmartScreen 경고가 표시되면 파일 출처가 GitHub Releases인지 확인한 뒤 추가 정보에서 실행할 수 있습니다.

Step 2

Gemini API 키 설정

처음 시작할 때는 Google AI Studio에서 Gemini API 키를 발급받아 Gemini 제공자로 등록하는 흐름을 권장합니다. API 키는 앱 설정 저장소에 로컬로 저장됩니다.

  1. Google AI Studio API Keys 페이지를 엽니다.
  2. Google 계정으로 로그인한 뒤 새 Gemini API 키를 만듭니다.
  3. TransNovel에서 설정 탭을 열고 LLM 섹션으로 이동합니다.
  4. 제공자 추가에서 Gemini를 선택하고, 발급받은 API 키를 붙여 넣어 저장합니다.
  5. 모델 추가에서 방금 만든 Gemini 제공자를 선택하고 사용할 모델을 저장합니다.
  6. 등록한 Gemini 모델을 활성 모델로 선택합니다.
API 키는 비밀번호처럼 다루세요. 공개 저장소, 이슈, 채팅, 스크린샷에 노출하지 않는 것이 안전합니다.
제공자 필요한 값 메모
Gemini Google AI Studio API key, model Quick Start 권장 경로입니다.
OpenRouter API key, model 여러 모델을 한 제공자로 테스트하기 쉽습니다.
OpenAI / Anthropic / Custom API key, base URL, model OpenAI-compatible Chat Completions 형식을 사용합니다.
OpenAI Codex 브라우저 로그인 앱에서 OAuth 로그인을 완료한 뒤 모델을 선택합니다.

다른 제공자를 쓰고 싶다면 OpenRouter, OpenAI-compatible 엔드포인트, OpenAI Codex 로그인도 같은 LLM 화면에서 추가할 수 있습니다.

Step 3

첫 챕터 번역

  1. 번역 탭으로 이동합니다.
  2. 지원 사이트의 챕터 URL을 입력합니다.
  3. 파싱 결과로 제목과 원문 문단이 보이면 번역을 실행합니다.
  4. 번역 결과는 문단 단위로 스트리밍됩니다. 완료 전에도 앞부분부터 읽을 수 있습니다.
  5. 결과가 마음에 들면 TXT 또는 HTML로 저장합니다.

지원 URL

현재 Syosetu, Hameln, Kakuyomu, Nocturne을 지원합니다. Kakuyomu는 개별 화 번역 중심으로 사용하는 편이 안정적입니다.

Step 4

관심작품 등록

자주 읽는 작품은 관심작품에 등록해 두면 새 글과 읽은 상태를 한 화면에서 확인할 수 있습니다.

  1. 관심작품 탭에서 작품 URL을 등록합니다.
  2. 새 화 여부와 챕터 목록을 확인합니다.
  3. 읽을 화를 선택한 뒤 번역 탭에서 개별 챕터를 번역합니다.
긴 챕터 번역은 모델 비용과 제공자 제한의 영향을 받습니다. 먼저 1-2개 챕터로 품질과 속도를 확인하세요.

Troubleshooting

자주 막히는 지점

번역 시작 시 API 키 오류가 납니다
설정의 LLM 제공자에 API 키가 저장되어 있는지, 활성 모델이 해당 제공자를 참조하는지 확인하세요.
URL을 넣어도 파싱되지 않습니다
지원 사이트의 작품 목록 URL이 아니라 개별 챕터 URL인지 확인하세요. 사이트 레이아웃 변경으로 파서가 깨졌다면 GitHub Issues에 URL과 오류 메시지를 남겨 주세요.
Nocturne 페이지가 열리지 않습니다
Nocturne은 성인 확인 쿠키가 필요한 사이트입니다. 앱이 자동 쿠키를 붙이지만, 사이트 정책이나 페이지 상태에 따라 실패할 수 있습니다.
번역 품질이 기대와 다릅니다
설정에서 시스템 프롬프트, 번역 메모, 치환 규칙을 조정한 뒤 같은 짧은 챕터로 다시 테스트하세요.

다음 단계

첫 번역이 끝났다면 관심작품 등록, 번역 캐시, API 로그, 치환 규칙을 차례로 확인해 보세요.
